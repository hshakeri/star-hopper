# Star Hopper — Expert Review & Improvement Roadmap (June 2026)

A multi-dimensional review of the game across gameplay, story, replayability, addictivity,
physics/science, comic-book art, code health, accessibility, and audio — followed by the
first implemented improvement wave and a prioritized backlog for what's next.

## Wave 1 — shipped (this pass)

All changes are zero-dependency vanilla JS, regression-tested (`node scratch/node-tests.js`,
58/58) and render-smoke-tested (`node scratch/node-smoke.js`). Cache version bumped to `coach-v48`.

1. **Boot fix (critical).** The main loop only ran `draw()` in the `playing` state, so the
   start screen showed a blank canvas. The loop now paints in the `start` state, and
   `draw()` has a start-state branch that renders the parallax starfield plus a new
   `drawStartBackdrop()` (home-planet horizon + moon, comic ink rims). — `game.js`
2. **Physics honesty.** Removed the gravity-by-character-type scaling (Rover ×0.7 / Hopper
   ×1.3) in `entities.js` and `physics.js` (sim, trajectory preview, and force-vector
   overlay) — both suits now free-fall at the same rate (Galileo). The heavy-vs-floaty feel
   is preserved through the mass-honest channels already present (`jump = jumpPower/mass`,
   `speed = engine/mass`). Two regression tests added (`test-runner.js`). Mass tooltip and a
   couple of antigravity strings reworded to avoid "cancel gravity" framing.
3. **Game-feel.** Coyote time (6-frame grace after leaving a ledge) plus a 6-frame jump buffer
   for just-before-landing inputs in `Player.update`, and wired audio: a soft landing chirp, a
   distinct rocket whoosh (replacing a probabilistic
   `playJump`), the previously-dead `playMagnet()`, and a new `playPortalUnlock()` fanfare on
   level clear. — `entities.js`, `audio.js`, `game.js`
4. **Comic pass.** Thick dark ink outlines on the Rover/Hopper silhouettes and gems; a new
   bold "impact pop-text" style (`ComicBubbles.pop`, capped at 3 concurrent) fed by the
   existing onomatopoeia pools, triggered on gem grab, enemy stomp, and mob kills. — `entities.js`, `game.js`
5. **Replayability / addictivity.** Locked-gem feedback (red pulse + wobble + throttled buzz)
   on every bump, not just the first; and the final-clear screen now points into the Daily
   Signal loop (new "TODAY'S SIGNAL" CTA) instead of dead-ending on "three worlds inbound". — `entities.js`, `game.js`, `index.html`

## Wave 2 — shipped (Depth, partial)

Cache version `coach-v49`. Tests: `node scratch/node-tests.js` → 62/62 (4 new). Verified
start screen + streak chip via headless Chrome.

1. **Deeper replay variety.** Added a `rvShiftTilesV` vertical-shift helper and new
   constraint-free geometry/target flavors so **Jupiter and Mag-Net go from 1 flavor (identical
   retries) to 3 each**, and Earth→4 / Moon→4 / Glacies→3. All preserve the required-gem count
   via the existing safety net (regression-tested over a deep attempt range). — `retry-variants.js`
2. **Gentle return-streak.** Tracks consecutive-day visits (`lastPlayedDate`/`streakCount`),
   increments on a new day, **silently resets after a gap (never shames)**, and shows a
   celebratory 🔥 chip on the start screen. Computed in `shApplyProgress` (after the saved
   fields load, since `Game.init` runs first). Consecutive-day rewards now also pop a small
   in-level streak cue, particles, and a Daily Lab CRT line when the cadet is in scene, so the
   habit loop has the same game feel as code and proof rewards. — `game.js`, `profiles.js`,
   `index.html`, `style.css`
3. **Animated planet-unlock.** Clearing a world plays a burst animation on the next galaxy-map
   node (`unlockNextPlanetNode`, respects `prefers-reduced-motion`). — `game.js`, `style.css`
4. **Persistence scaffolding.** One additive, backward-compatible profile schema for all
   Phase-2 fields (`bestClearTimes`, `masteryCleared`, `masteryMeters`, `dailySignalClears`,
   `lastPlayedDate`, `streakCount`) wired once through `shCaptureProgress`/`shApplyProgress`;
   kept OUT of the github-sync flat payload (local profile is authoritative). — `profiles.js`, `game.js`
